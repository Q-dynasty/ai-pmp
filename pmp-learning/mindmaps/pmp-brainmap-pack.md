# PMP 纯脑图学习包

这份脑图包是新的核心学习入口。

使用原则：

- 先看总脑图，建立主线
- 再看核心知识图，把 PMBOK 7 和 PMBOK 6 串起来
- 再看高频易错图，训练做题判断
- 最后刷口诀记忆图，用于考前冲刺

## PMP 总脑图
> slug: pmp-master-map
> eyebrow: Master Brainmap
> description: 先搭全局骨架，再进入核心知识、高频判断和记忆冲刺。

```mermaid
mindmap
  root((PMP 速通总图))
    学习顺序
      先总后分：先搭骨架再补细节
      先理解再应试：第七版讲为什么
      最后冲刺：高频判断和口诀反复刷
    核心知识
      PMBOK 7：原则和绩效域决定管理思路
      PMBOK 6：过程组和知识领域决定考试框架
      敏捷：迭代交付响应变化重协作
    高频判断
      变更控制：所有变更先评估再批准
      风险和问题：未发生看风险已发生看问题
      管质量和控质量：改过程与查结果要分清
      沟通和干系人：传信息与管参与不是一回事
    记忆抓手
      启规执监收：五大过程组顺序
      整范进成质资沟风采干：十大知识领域
      风险口诀：先规后识再定再量跟应对再监督
      干系人口诀：先识别再规划后管理再监督
    视频精华
      先搭框架再背细节：零基础最省力
      第七版先理解价值团队系统思维
      冲刺题先分场景再选流程动作
```

## 核心知识图
> slug: core-knowledge-map
> eyebrow: Core Knowledge Map
> description: 把 PMBOK 7 的理解主线、PMBOK 6 的应试主线和敏捷核心串成一张图。

```mermaid
mindmap
  root((核心知识图))
    PMBOK 7
      12 原则：做决策的价值准绳
        聚焦价值：不是只交付而是交付有用结果
        团队协作：高绩效团队比死守流程更重要
        系统思维：局部变化要看整体影响
        质量内建：预防问题比返工更值钱
        适应与韧性：计划可调整目标别丢
        变革引领：变化是常态要主动引导
      8 绩效域：看项目是否健康运转
        干系人：先识别谁关键再设计参与方式
        团队：授权协作反馈比单点英雄更重要
        开发方法与生命周期：先选适合的做法
        规划：滚动规划不是一次写死
        项目工作：交付推进和团队协同一起管
        交付：结果必须可用可验收
        度量：用数据发现偏差
        不确定性：风险和机会都要管
    PMBOK 6
      五大过程组：项目从启动到收尾的主流程
        启动：立项和识别关键人
        规划：把目标拆成可执行方案
        执行：按计划产出成果
        监控：发现偏差及时纠正
        收尾：正式结束并沉淀经验
      十大知识领域：考试最常见框架
        整合：把全局统起来
        范围：做什么不做什么
        进度：什么时候做完
        成本：预算和花费怎么控
        质量：过程变优结果合格
        资源：人材设备怎么配
        沟通：信息要准要及时
        风险：先预判再应对
        采购：外部资源怎么买
        干系人：关键人物怎么稳住
    敏捷核心
      价值优先：先交付最有价值的东西
      小步快跑：短迭代持续反馈
      欢迎变化：变化不是敌人
      团队自治：靠协作不是靠命令
    视频精华
      第七版负责为什么这样管项目
      第六版负责考试通常怎么考
      敏捷题多看团队反馈客户价值变化响应
```

## 高频易错图
> slug: exam-pitfall-map
> eyebrow: Exam Pitfall Map
> description: 这张图专门压缩最常考、最易混、做题最需要快速判断的场景抓手。

```mermaid
mindmap
  root((高频易错图))
    变更控制
      核心抓手：所有正式变更先走整体变更控制
      做题提醒：别一着急就直接改计划或改范围
      视频精华：题里只要出现变更先想评估和审批
    范围管理
      WBS：把范围拆清才谈进度成本
      范围基准：后面确认和控制都靠它
      易错提醒：需求收集不等于范围确认
    进度与成本
      关键路径：决定最短工期
      进度压缩：赶工和快速跟进要会分
      挣值判断：CPI 看花钱 SPI 看进度
      记忆抓手：CPI 小于 1 超支 SPI 小于 1 落后
    风险判断
      风险：还没发生要提前准备
      问题：已经发生要立刻处理
      风险流程：先规后识再定再量跟应对再监督
      视频精华：场景题先判断发生没发生
    质量判断
      管理质量：改过程让以后少错
      控制质量：查结果看现在合不合格
      口诀：管过程控结果
    沟通与干系人
      沟通管理：信息传递是否准确及时
      干系人管理：关键人物是否支持项目
      做题提醒：看题干是在传信息还是稳关系
    合同采购
      总价合同：范围清晰时更稳
      成本补偿：范围不清先做再结算
      工料合同：短期灵活但要盯成本
```

## 口诀记忆图
> slug: memory-map
> eyebrow: Memory Map
> description: 把最需要强记的顺序、公式和判断口令压缩到一张图里，适合考前高频刷。

```mermaid
mindmap
  root((口诀记忆图))
    五大过程组
      启规执监收：先立项再定法边做边看最后收
      考试抓手：先问题目处在哪个阶段
    十大知识领域
      整范进成质资沟风采干
      记忆方式：先背简称再反推全称
    风险流程
      先规后识
      再定再量
      跟着应对
      最后监督
    干系人流程
      先识别
      再规划
      后管理
      再监督
    挣值公式
      PV：计划做到哪
      EV：实际挣到哪
      AC：实际花多少
      CPI：EV除AC看值不值
      SPI：EV除PV看赶没赶上
    易混口诀
      管过程控结果
      风险未发生问题已发生
      沟通传信息干系人管关系
    视频精华
      冲刺阶段只背高频主干别贪全
      每背一句都要能反推原概念
      先看场景关键词再套流程动作
```
