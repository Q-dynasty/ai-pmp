# PMP 纯脑图学习包重构设计

## 背景

当前站点已经具备：

- 在线学习首页
- 多篇 Markdown 学习文档
- 3 张 Mermaid 脑图
- GitHub Pages 在线托管

但当前形态仍以“导航式学习”与“视频摘要列表”为主，不完全符合新的目标。

## 新目标

新的成品需要满足以下要求：

- 核心知识点直接整合到脑图中，而不是靠多个文档来回跳转
- 每个知识点要有足够清晰的白话讲解
- 视频内容不再以链接列表呈现，而是提炼为可复习的结论
- 记忆辅助直接内嵌到脑图结构里
- 以一个核心 Markdown 源文件作为事实来源，再派生出多个在线脑图页面

## 设计决策

采用“单一主源文件 + 多张纯脑图页面”的方案。

### 主源文件

新增一个核心 Markdown 文件：

- `pmp-learning/mindmaps/pmp-brainmap-pack.md`

该文件作为脑图内容的唯一维护入口，内部包含多个脑图分区：

1. 总脑图
2. 核心知识图
3. 高频易错图
4. 口诀记忆图

每个分区都直接包含：

- 知识点名
- 白话解释
- 考试抓手
- 易混提醒
- 口诀记忆
- 视频精华结论

### 页面形态

在线端保留 1 个入口页 + 4 张脑图页：

- 脑图总入口
- PMP 总脑图
- 核心知识图
- 高频易错图
- 口诀记忆图

页面不再强调“查看源文件”，避免用户继续走“看文档链接”的学习路径。

### 视频内容处理

原“视频精选”不再作为主学习入口，而是把提炼结果内化到脑图节点中，例如：

- “第七版先理解价值，再记流程”
- “场景题先分风险和问题”
- “管理质量看过程，控制质量看结果”

保留研究记录文件仅用于资料追溯，不在主学习路径中突出展示。

## 构建设计

现有脑图构建脚本需要从“单文件单脑图”升级为“单文件多脑图”：

- 解析一个 Markdown 中多个 `##` 分区
- 读取每个分区中的标题、元信息和 Mermaid 图块
- 生成对应 HTML 页面
- 刷新脑图总入口页

## 验证策略

实现完成后需要验证：

- 新的脑图包解析测试通过
- 构建脚本能从单一主源文件生成全部脑图页面
- 校验脚本覆盖新的主源文件和 4 张脑图页
- 在线首页与脑图入口不再以视频链接作为主学习路径
