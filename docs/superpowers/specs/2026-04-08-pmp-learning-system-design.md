# PMP 30 天学习系统设计文档

## 1. 背景与目标

当前工作区包含两本核心教材：

- `PMBOK第六版_中文版（带完整目录）.pdf`
- `PMBOK第七版（中文版）.pdf`

目标是围绕这两本教材，为零基础学习者构建一套可直接使用的 PMP 学习系统，满足以下需求：

- 整理教材内容，形成可持续复习的结构化笔记
- 筛选 Bilibili 上高分、口碑稳定的 PMP 视频课程
- 对筛选出的视频做结构化摘要，帮助快速学习
- 生成一个高效、可本地打开的在线学习文档
- 同时生成一套配套 Markdown 知识库，便于长期沉淀
- 输出思维导图，帮助建立整体框架
- 将复杂、难记、易混内容整理为顺口溜和记忆卡

用户画像与学习约束如下：

- 基础：零基础
- 周期：30 天
- 日学习时长：2-4 小时
- 成品形态：网页 + Markdown 双交付

## 2. 设计原则

本项目采用“双主线融合型”设计。

### 2.1 学习主线

第一主线使用 PMBOK 第七版建立认知骨架：

- 12 项原则
- 8 个绩效域

目的不是先背概念，而是先理解项目管理整体在解决什么问题。

第二主线使用 PMBOK 第六版建立考试框架：

- 5 大过程组
- 10 大知识领域
- 49 个过程

目的是把理解转化为应试结构，降低流程、概念、场景题的混乱感。

### 2.2 内容表达原则

每个重点主题都尽量按四层结构输出：

1. `先理解`：用白话解释概念和作用
2. `再结构化`：用表格、框架、对比梳理内容
3. `再记忆`：补充口诀、速记卡、脑图
4. `再复习`：说明适合在哪个阶段回看

### 2.3 用户体验原则

- 零基础友好，尽量少依赖默认术语背景
- 学习阶段和冲刺阶段的内容分层展示
- 网页版与 Markdown 版内容结构保持一致
- 重点突出“看得懂、翻得快、背得住”

## 3. 交付范围

### 3.1 在线学习网页

产出一个本地可直接打开的单页学习网页，作为学习入口与总导航。网页应包含以下模块：

- 学习总览
- 30 天学习路径
- PMBOK 第七版认知主线
- PMBOK 第六版应试主线
- 高频考点速查
- Bilibili 视频精选与摘要
- 口诀与记忆强化区
- 思维导图入口或内嵌展示
- 最后 7 天冲刺复盘区

网页应满足以下要求：

- 纯本地可访问，不依赖在线服务才能查看
- 桌面端与移动端均可阅读
- 适合长文学习，具备清晰的导航、目录和分区
- 风格简洁但不呆板，重点信息具备明显层级

### 3.2 Markdown 知识库

产出一套与网页一致的 Markdown 知识库，建议目录如下：

- `pmp-learning/00-学习总览.md`
- `pmp-learning/01-30天计划.md`
- `pmp-learning/02-PMBOK7-12原则.md`
- `pmp-learning/03-PMBOK7-8绩效域.md`
- `pmp-learning/04-PMBOK6-五大过程组.md`
- `pmp-learning/05-PMBOK6-十大知识领域.md`
- `pmp-learning/06-49过程速记.md`
- `pmp-learning/07-B站视频精选与摘要.md`
- `pmp-learning/08-高频考点与易错点.md`
- `pmp-learning/09-口诀与记忆卡.md`
- `pmp-learning/10-思维导图.md`

必要时可补充静态资源目录，例如：

- `pmp-learning/assets/`
- `pmp-learning/mindmaps/`

### 3.3 思维导图

至少输出 3 张思维导图，优先使用文本可维护格式生成，例如 Mermaid。

必备导图：

1. `PMP 全局总图`
   - 展示第七版认知框架与第六版考试框架的对应关系
2. `第六版考试框架图`
   - 展示 5 大过程组、10 大知识领域、49 过程的整体结构
3. `冲刺记忆图`
   - 聚焦高频考点、易混点、口诀与速记内容

### 3.4 冲刺资料

单独整理一层考前速看资料，便于后期高频复盘。至少包括：

- 高频考点清单
- 易混淆概念对比
- 关键流程速记
- 公式与逻辑速记
- 顺口溜与口诀
- 最后 7 天复盘建议

## 4. 教材整理策略

### 4.1 第七版整理方式

第七版承担“建立认知”的作用，整理重点如下：

- 12 项原则的核心含义
- 8 个绩效域分别解决什么问题
- 原则与绩效域的联系
- 第七版对现代项目管理思维的强调点

输出时重点回答：

- 这个概念是什么
- 为什么重要
- 它和实际项目管理有什么关系
- 它与第六版的流程化表达如何对应

### 4.2 第六版整理方式

第六版承担“建立应试结构”的作用，整理重点如下：

- 5 大过程组的前后逻辑
- 10 大知识领域的职责边界
- 49 过程的分布与联系
- 高频输入、工具与技术、输出的考试规律
- 场景题中常见的判断逻辑

输出时重点回答：

- 这一过程/领域在项目生命周期中处于什么位置
- 与前后过程的衔接关系是什么
- 常见考法是什么
- 什么地方最容易记混
- 有什么更快的记忆方式

### 4.3 融合输出方式

同一主题尽量建立“第七版理解角度”和“第六版考试角度”的对应关系，避免形成两套完全割裂的知识。

示例：

- 第七版讲“价值、团队、系统思维”
- 第六版落到“范围、进度、成本、风险、干系人”等过程化管理

最终输出要帮助学习者形成如下认知：

- 第七版告诉我项目管理为什么这样做
- 第六版告诉我考试和实践中通常怎么落地

## 5. 视频研究与摘要方法

### 5.1 研究目标

为学习者筛选 Bilibili 上适合零基础学习 PMP 的高质量视频，并给出清晰的使用建议。

### 5.2 视频筛选维度

实现阶段需基于 Bilibili 的实时可获取结果进行筛选，优先考虑以下维度：

- 播放量
- 收藏量
- 点赞量
- 评论口碑
- 更新时间
- 是否为系统课程或成套内容
- 是否更适合零基础
- 是否更适合框架梳理
- 是否更适合冲刺复盘

### 5.3 视频产出格式

每个入选视频或课程条目至少输出：

- 标题
- 链接
- 推荐理由
- 适合观看阶段
- 覆盖主题
- 内容摘要
- 建议重点记忆内容
- 与教材的对应关系

视频列表至少分成三类：

1. `入门类`
   - 帮助建立整体印象
2. `系统课类`
   - 帮助搭框架、系统学习
3. `冲刺类`
   - 帮助快速复盘与查漏补缺

### 5.4 视频摘要写法

摘要不做机械转录，而是做结构化提炼。优先采用以下模板：

- 这节视频主要讲了什么
- 适合在学习的哪一阶段看
- 视频里的重点概念是什么
- 哪些内容值得记忆或反复看
- 它与 PMBOK 第六版/第七版的对应点是什么

## 6. 口诀与记忆系统设计

口诀只覆盖“复杂、易混、需要强记”的内容，不追求为所有知识点强行编口诀。

优先覆盖主题：

- 5 大过程组顺序
- 10 大知识领域框架
- 49 过程分布记忆
- 变更管理流程
- 风险管理流程
- 采购与合同常见考点
- 挣值管理公式与逻辑
- 易混概念对比

口诀设计标准：

- 短
- 顺
- 能唤起结构
- 不为了押韵牺牲准确性

除口诀外，可补充：

- 速记卡
- 易混点对照表
- 公式记忆块

## 7. 30 天学习路径设计

学习计划按四个阶段组织：

### 阶段一：建立整体认知（第 1-5 天）

重点：

- 认识 PMP 考试和学习路径
- 熟悉 PMBOK 第七版框架
- 建立“原则 + 绩效域”的整体地图

目标：

- 能说出项目管理整体在管什么
- 对术语不过度恐惧
- 具备继续进入第六版的认知支架

### 阶段二：系统攻克考试框架（第 6-18 天）

重点：

- 学习五大过程组
- 学习十大知识领域
- 理解 49 过程的结构位置

目标：

- 能把过程组、知识领域、过程之间的关系串起来
- 知道高频考点分布

### 阶段三：重点强化（第 19-24 天）

重点：

- 高频考点
- 易错点
- 视频查漏补缺
- 口诀强化记忆

目标：

- 缩小知识盲区
- 提高识别题目意图的能力

### 阶段四：冲刺复盘（第 25-30 天）

重点：

- 脑图串联
- 高频清单回看
- 口诀复盘
- 冲刺资料速看

目标：

- 在有限时间内高频复现核心框架
- 将记忆从“散点”变成“网络”

## 9. 风险与处理策略

### 9.1 PDF 可提取性风险

当前本地环境未确认可直接稳定抽取 PDF 文字内容。实现阶段如遇提取困难，可采用以下策略：

- 先依据稳定知识框架构建学习系统骨架
- 再结合可提取片段、人工梳理与外部公开视频进行校准
- 对无法直接抽取的部分，不伪造教材引用页码

### 9.2 视频时效性风险

Bilibili 视频排序、热度、可见性会变化。实现阶段需以实时检索结果为准，并在资料中标注筛选日期。

### 9.3 信息过载风险

零基础学习者容易被过多术语压垮。实现阶段要控制每页信息密度，并明确区分：

- 必学
- 选学
- 冲刺看

## 10. 验收标准

最终成品应满足以下标准：

- 零基础用户能按 30 天路径顺序学习
- 网页版与 Markdown 版结构一致
- 资料同时支持“系统学习”和“考前速看”
- 视频列表不只是罗列链接，而是给出阶段建议和摘要
- 至少产出 3 张结构清晰的思维导图
- 对复杂、易混内容给出有助记忆的口诀或速记法
- 内容整体清晰说明第七版与第六版之间的联系

### 10.1 验证方式

实现完成后，至少执行以下检查：

- 打开 `pmp-learning/index.html`，确认本地离线可正常浏览
- 检查网页在常见桌面宽度和窄屏宽度下的可读性
- 抽查网页与 Markdown 中的模块标题，确认结构一致
- 抽查视频条目，确认包含链接、推荐理由、摘要、适看阶段
- 抽查思维导图，确认能与正文内容相互对应
- 抽查口诀内容，确认没有为追求押韵而扭曲概念
- 在最终资料中标明 Bilibili 视频筛选日期，避免时效性误导

## 11. 非目标

本项目当前阶段不包含以下内容：

- 模拟题库系统开发
- 在线账号同步
- 视频下载或转载
- 将全部教材做逐页、逐段完整转录
- 生成未经核实的“官方题库答案”

## 12. 当前已知约束与假设

- 当前目录不是 Git 仓库，因此本阶段无法按常规要求提交设计文档 commit
- 视频研究需要在实现阶段进行实时检索，不能依赖过期记忆
- 成品优先保证可学习性与结构质量，不追求复杂技术实现
